# Naming Conventions

## Follow the Domain

The domain has a very expressive language to explain every detail. Refer to the
[Domain Model](../domain-model/README.md) for what language to use.

## Naming Tests

Test names should be human-readable and form a coherent narrative. Tests tell a
story for humans. That excludes any technical jargon.

Execption: When the test names are purely technical/utility

## Vocabulary

CRUD - Create, Read, Update or Delete for comments or function names are bad and
banned words

Exception: When working on database queries
