Blueprint for schemas used to document endpoints in JSONAPI.

## Prerequisite

- The [Domain Model](../../../../domain-model/README.md) contains the command
  rsp. query/read model
- The [Public API](../domain/public-api.md) has `@spec` annotations for the operation

## Implementation

Represents OpenAPI schemas to document the endpoint.

- Use `OpenApiSpex`
- Schemas per endpoint:
  - **Query (single)**: Response schema + Query parameter schema
  - **Query (collection)**: Response schema + Query parameter schema
  - **Command**: Request schema + Response schema
- The schema is referenced by the `operation` in the controller
- There is no need to register the schema in API spec. Using them in the controller is enough.

### Schema Derivation

Schemas are derived from the **Public API** `@spec` annotations, the **View**, and the **Domain Model**:

- **Request Schema** (commands only): Derived from the parameter type in the `@spec`
- **Response Schema**: Derived from the return type in the `@spec`
- **Query Parameter Schema** (queries): Derived from the view's fields and the domain model's relationships/filterable/sortable fields

#### Return Type Mapping

| Public API Return Type | Response Schema |
|------------------------|-----------------|
| `Architecture.public_api(<DomainObject>ReadModel.t())` | Single object response |
| `Architecture.public_api([<DomainObject>ReadModel.t()])` | Collection response |
| `Architecture.public_api()` | No response schema (DELETE commands) |

#### Parameter Type Mapping (Commands)

The command's parameter map in the `@spec` defines the request schema:

```elixir
# Public API @spec:
@spec suggest_sport(%{
  required(:title) => String.t(),
  required(:slug) => String.t(),
  optional(:description) => String.t() | nil
}) :: Architecture.public_api(SportReadModel.t())
```

Maps to:

- `title`: required string property
- `slug`: required string property
- `description`: optional string property (nullable)

**Special case for CREATE commands**: The `:id` field is auto-generated by the public API and should NOT be included in the request schema.

#### Query Parameter Mapping (Queries)

Query parameter schemas document the available query options:

**For single record queries:**

- **fields**: Derive from the view's `fields/0` function
- **include**: Derive from the domain model's relationships

**For collection queries:**

- **filter**: Derive from the domain model's filterable fields
- **sort**: Derive from the domain model's sortable fields
- **page**: Standard pagination parameters (number, size)
- **fields**: Same as single record
- **include**: Same as single record

### Documentation

Derive all documentation from the [Domain Model](../../../../domain-model/README.md):

- **`@moduledoc`**: Describe the schema's purpose for documenting the endpoint.
- **`@doc`**: Describe the `schema/0` function.

Example:

```elixir
@moduledoc """
OpenAPI schema for documenting sport endpoints.
"""

@doc """
Returns the OpenAPI schema for the sport response.
"""
```

### Implementation Template

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<Operation>Response do
  @moduledoc """
  OpenAPI schema for documenting <domain_object> endpoints.
  """

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <operation> response.
  """
  OpenApiSpex.schema(%{...})
end
```

### Example: Single Object Response Query

A query returning a single read model will use this template
Using the [view](./view.md).

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<DomainObject>Response do
  @moduledoc """
  OpenAPI schema for documenting <domain_object> endpoints.
  """

  alias SportipediaWeb.<Subdomain>.<Composite>.<DomainObject>View
  alias Sportipedia.Support.JSONAPI.OpenApiSchema

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <domain_object> response.
  """
  OpenApiSpex.schema(OpenApiSchema.from_view(<DomainObject>View, title: "<composite>.<DomainObject>"))
end
```

### Example: Collection Response Query

A query returning a read model collection will use this template.
Using the [view](./view.md).

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<DomainObject>CollectionResponse do
  @moduledoc """
  OpenAPI schema for documenting <domain_object> collection endpoints.
  """

  alias SportipediaWeb.<Subdomain>.<Composite>.<DomainObject>View
  alias Sportipedia.Support.JSONAPI.OpenApiSchema

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <domain_object> collection response.
  """
  OpenApiSpex.schema(
    OpenApiSchema.from_view(<DomainObject>View, title: "<-composite>.<DomainObject>s", many: true)
  )
end
```

### Example: Single Record Query Parameters

A single record query needs a query parameter schema documenting fields and includes.

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<Operation>QueryParams do
  @moduledoc """
  OpenAPI schema for documenting <operation> query parameters.
  """

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <operation> query parameters.
  """
  OpenApiSpex.schema(%{
    title: "<composite>.<Operation>QueryParams",
    type: :object,
    properties: %{
      fields: %OpenApiSpex.Schema{
        type: :object,
        properties: %{
          <domain_object>: %OpenApiSpex.Schema{
            type: :string,
            description: "Comma-separated fields to include in the response",
            enum: [<# fields from view's fields/0>]
          }
        }
      },
      include: %OpenApiSpex.Schema{
        type: :string,
        description: "Comma-separated relationships to side-load",
        enum: [<# relationships from domain model>]
      }
    }
  })
end
```

### Example: Collection Query Parameters

A collection query needs a query parameter schema documenting filter, sort, page, fields, and includes.

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<Operation>QueryParams do
  @moduledoc """
  OpenAPI schema for documenting <operation> collection query parameters.
  """

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <operation> collection query parameters.
  """
  OpenApiSpex.schema(%{
    title: "<composite>.<Operation>QueryParams",
    type: :object,
    properties: %{
      filter: %OpenApiSpex.Schema{
        type: :object,
        properties: %{
          # filterable fields from domain model
          status: %OpenApiSpex.Schema{type: :string, enum: ["suggested", "approved", "archived"]}
        }
      },
      sort: %OpenApiSpex.Schema{
        type: :string,
        description: "Comma-separated fields to sort by. Prefix with `-` for descending.",
        enum: ["title", "-title", "inserted_at", "-inserted_at"]
      },
      page: %OpenApiSpex.Schema{
        type: :object,
        properties: %{
          number: %OpenApiSpex.Schema{type: :integer, default: 1},
          size: %OpenApiSpex.Schema{type: :integer, default: 20}
        }
      },
      fields: %OpenApiSpex.Schema{
        type: :object,
        properties: %{
          <domain_object>: %OpenApiSpex.Schema{
            type: :string,
            description: "Comma-separated fields to include in the response",
            enum: [<# fields from view's fields/0>]
          }
        }
      },
      include: %OpenApiSpex.Schema{
        type: :string,
        description: "Comma-separated relationships to side-load",
        enum: [<# relationships from domain model>]
      }
    }
  })
end
```

### Example: Command Request Schema

A command request schema is derived from the public API's parameter `@spec`.

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.Schemas.<Command>Request do
  @moduledoc """
  OpenAPI schema for documenting <command> request parameters.
  """

  require OpenApiSpex

  @doc """
  Returns the OpenAPI schema for the <command> request.
  """
  OpenApiSpex.schema(%{
    title: "<composite>.<Command>Request",
    type: :object,
    properties: %{
      title: %OpenApiSpex.Schema{type: :string, description: "The title"},
      slug: %OpenApiSpex.Schema{type: :string, description: "The slug"},
      description: %OpenApiSpex.Schema{type: :string, nullable: true, description: "Optional description"}
    },
    required: [:title, :slug]
  })
end
```

### Example: Command Response Schema

A command resulting in a CREATE or UPDATE returns the read model.
Use the same template as "Single Object Response Query".

A command resulting in a DELETE has no response schema.
Use `no_content` in the controller's operation definition.

## Test

Use `use ExUnit.Case` (no DB, no sandbox). Test that the compile-time schema has the expected structure:

### Test Template

```elixir
defmodule SportipediaWeb.<Subdomain>.<Composite>.<DomainObject>SchemaTest do
  use ExUnit.Case

  alias SportipediaWeb.<Subdomain>.<Composite>.Schemas.<DomainObject>Response

  describe "<DomainObject>Response" do
    test "schema/0 has the correct title" do
      assert %{title: "<_composite>.<DomainObject>"} = <DomainObject>Response.schema()
    end

    test "schema has data with id, type, and attributes" do
      schema = <DomainObject>Response.schema()
      data_props = schema.properties.data.properties
      assert Map.has_key?(data_props, :id)
      assert Map.has_key?(data_props, :type)
      assert Map.has_key?(data_props, :attributes)
    end
  end
end
```
